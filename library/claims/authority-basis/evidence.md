# Evidence Capture

Capture evidence for this claim type in the shared evidence shape: claim,
context or method, observations and raw artifacts, verdict.

## Claim

- State the concrete claim using the rules in `claim.md`.
- Name the authoritative source, definition, or prior decision.
- State the topic and limit of the authority choice.

## Methodology References

- Record the claim type folder path.
- Link relevant `canonical-sources.md`, `decision-ledger.md`, and `plan.md`
  sections.
- Link receipts or edited pages where conflicting material was aligned.

## Method Or Context

- Explain why authority needed to be established or clarified.
- Record how the run compared competing sources or definitions.
- Note whether the resolution came from an existing canonical source, a fixed
  decision, or a new authority choice introduced during the pass.

## Observations

- Record the competing sources, definitions, or pages.
- Record why the chosen authority was treated as controlling.
- Record what downstream material was aligned, linked, narrowed, or left open.
- Record any remaining unresolved authority conflict.

## Raw Artifacts

- Reference canonical-source register entries and decision-ledger entries.
- Reference before and after page excerpts showing conflict resolution.
- Reference receipts for canonicalization, terminology alignment, or
  traceability work.

## Verdict

- Use the verdict meanings defined below.
- Base the verdict on whether the authority basis is explicit, inspectable, and
  reflected in the resulting material.
- Note the limiting factor when authority remained partial or contested.

## Open Questions Or Follow-Ups

- Record unresolved source conflicts or missing owner confirmation.
- Record follow-up when an authority choice needs formal ratification.
- Include revisit triggers when the chosen authority may change.

## Verdict Guidance

Define what each allowed verdict means for this claim type.

### supported

- Use when: the authority basis is explicit, inspectable, and the aligned
  material reflects it consistently.
- Not enough when: one source was used in practice but never recorded as
  authoritative.

### mixed

- Use when: the main authority is clear but some neighboring material remains
  only partially aligned or one source conflict remains open.
- Not enough when: the central authority choice itself is still unclear.

### incomplete

- Use when: the run implicitly relied on authority but did not record enough to
  justify later trust.
- Not enough when: contradictory authority choices were already applied.

### open

- Use when: competing authority bases remain under evaluation or owner input is
  pending.
- Not enough when: the run has already finalized a resolution while still
  claiming authority is unresolved.

### failed

- Use when: the run relied on a clearly unsupported authority basis or left
  material conflict unresolved while presenting it as settled.
- Not enough when: only minor traceability links are missing.

## Evidence Thread Rules

Create a new evidence item when:

- a different topic needs a different authority basis,
- a new source conflict is independent from the current one,
- one thread would otherwise mix source authority with change propagation.

### Enrich An Existing Evidence Item When

- more competing material is discovered for the same topic,
- more downstream alignment evidence is added,
- the same authority resolution is strengthened or qualified.

### Do Not Mix

- authority resolution for different topics,
- source authority with approval or ownership evidence,
- authority resolution with a separate diagnosis thread unless the diagnosis is
  strictly the same conflict.

### Close Out When

- the authority basis is explicit for the current topic,
- aligned or deferred downstream material is recorded,
- remaining conflict or fragility is captured explicitly.
