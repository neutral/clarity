# Step: normalize-headings

Purpose: Standardize heading levels and naming for scanability and consistency.

Inputs:
- Target set (paths/headings/labels)

Steps:
1) Check heading levels for a consistent hierarchy.
2) Rename headings to use parallel grammar and clear intent.
3) Merge or split headings when a section covers multiple intents.

Outputs:
- Consistent heading hierarchy and naming.
