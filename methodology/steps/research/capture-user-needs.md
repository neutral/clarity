# Step: capture-user-needs

Purpose: Make user and customer needs explicit for design and planning.

Inputs:
- Target set (paths/headings/labels)
- UX research notes or stakeholder input

Steps:
1) Identify key personas or customer segments.
2) Record user needs, pain points, and jobs to be done.
3) Add concrete use cases or scenarios when possible.

Outputs:
- User Need, Persona, and Use Case statements.
